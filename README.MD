# JusTran
Набор инструментов для точного и контролируемого перевода юридических текстов. (Рабочий прототип)

[Слайды про проблематику](https://disk.360.yandex.ru/i/Ez9LmyeBw8Nzbw)

## Функции
* Перевод
* Выдача релевантных вариантов перевода
* Лексикографическая справка

## Как это работает
Экземпляр JusTran принимает HTTP запросы согласно API. Авторизация пользователей проходит по логину и паролю в соответствии с базой данных. После авторизации для запросов используется `Bearer` токен. Для перевода особым образом промптится подходящая LLM[^1]. Клиенту возвращается перевод и найденные варианты перевода. При помощи клиента пользователь также может найти в глоссариях пояснения к вариантам перевода, помогающие выбрать верный.

[^1]: В этой версии — удалённая, хотя в ходе разработки использовались и локальные. В итоговом варианте продукта предполагается использование локальной модели.

## Как это запустить
Для запуска понадобится Python 3.12 и запущенный [PostgreSQL](https://www.postgresql.org/download/).

```bash
git clone https://github.com/vmgorelikov/justran.git
cd justran
python -m venv .venv
.venv/Scripts/activate
pip install -r requirements.txt
cd src
psql -U postgres_username -d postgres_database_name -f schemas/db_schema_postgresql.sql
```

В `.env_` нужно вставить используемые секреты и переименовать его в `.env`:
* `DATABASE_URL` — URL, по которому SQLAlchemy смог бы найти PostgreSQL базу данных.
* `JWT_SECRET` — секрет для выдачи и проверки JWT токенов при аутентификации. Может быть любым, но рекомендуется брать токен большой длины и с наибольшей энтропией.
* `OPENAI_API_KEY` — ключ для запросов к удалённой модели.

```bash
python main.py
```

## Структура репозитория

- `.env` Переменные окружения с секретами
- `data/` Лексикографические данные
- `src/` Код
+ - `api/` Описание и реализация REST API
+ - `schemas/` Схемы REST API
+ - `auth/` Логика авторизации
+ - `db/` Расширимая обёртка над `sqlalchemy.create_engine`.
+ - `models/` Описание модели данных
+ + - `lexical_db.py` для лексикографических данных,
+ + - `service_db.py` и
+ + - `db_schema_postgresql.sql` — для пользовательских данных.
+ - `core/` Основная логика
+ + - `translation.py` Получение перевода
+ + - `lexicography.py` Получение лексикографических данных
+ - `llm/` Работа с моделью-переводчиком
+ + - `translation.py` Чанкинг, промптинг, получение и обработка перевода и вариантов
+ + - `prompt_constructor.py` Промпты
+ + - `model_constructor.py` Расширимый код для создания интерфейса с моделью
+ - `tools/` Инструменты для модели-переводчика
+ + - `glossary_searcher.py` Поиск по глоссарию
+ + `frontend/` UI для демо
